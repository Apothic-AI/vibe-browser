# VIBE.md

## Service

Name: Local Vibe Browser Demo

Summary: This file exists so the browser can test the full Vibe discovery flow during local development.

Audience: Developers building agentic AI systems.

## Instructions

Render a developer-facing landing page for a fictional service called "Signal Garden".

The page should feel like a protocol preview, not a marketing splash page. Keep it clean, articulate, and tool-friendly.

The experience should emphasize:

- machine-readable publishing
- agent-native browsing
- low-friction discovery
- rendering from instructions instead of HTML templates

## Experience

Preferred tone: technical, calm, exact

Preferred layout:

- compact hero
- three capability cards
- short workflow section
- footer note explaining that this page was generated from VIBE.md

Preferred visual direction:

- warm paper background
- teal accent
- editorial spacing
- monospace details used sparingly

## Integrations

- None required for this demo.

## Notes

The browser should forward this entire file to the ACP agent and let the agent decide how to adapt it into a page.
